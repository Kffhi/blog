CSS布局之浮动

发表于 2019-05-16 | 分类于 前端 ， CSS

CSS布局（四）——浮动

一些概念

定义

摘自MDN：float CSS属性指定一个元素应沿其容器的左侧或右侧放置，允许文本和内联元素环绕它。该元素从网

页的正常流动(文档流)中移除，尽管仍然保持部分的流动性（与绝对定位相反）

简言之，浮动的本意就是让文字像流水一样环绕浮动元素

特性

包裹性

block元素不指定width的话，默认是100%，一旦让该div浮动起来，立刻会像inline元素一样产生包裹性，宽度会

跟随内容自适应。（这也是通常float元素需要手动指定width的原因）

破坏性

文字之所以会环绕含有float属性的图片时因为浮动破坏了正常的line boxes。

在CSS，所有的高度都是由两个模型产生的：

  • box盒模型 对应CSS为height+padding+margin，受height及img的height的值控制
  • line box 对应样式为line-height，而float而之所以能实现浮动环绕，就是因为它破坏了元素的inline boxes，元素已经没有了高度
关于文档流

常说的绝对完全脱离文档流，浮动不完全脱离文档流，而这造成的差异就是：同处于文档流的元素实体不会与浮动元素重叠，而会与绝对定位重叠。这就是文字围绕的重要原因之一：虽然浮动元素高度为零，但是因为宽度实体存在（包裹性），同样是content area 实体的文字不会与之重叠（外部的容器盒子containing box会重叠）

清除浮动
