CSS布局之浮动

发表于 2020-02-15 | 分类于 前端 ， CSS

CSS布局（四）——浮动

一些概念

定义

摘自MDN：float CSS属性指定一个元素应沿其容器的左侧或右侧放置，允许文本和内联元素环绕它。该元素从网

页的正常流动(文档流)中移除，尽管仍然保持部分的流动性（与绝对定位相反）

简言之，浮动的本意就是让文字像流水一样环绕浮动元素

特性

包裹性

block元素不指定width的话，默认是100%，一旦让该div浮动起来，立刻会像inline元素一样产生包裹性，宽度会

跟随内容自适应。（这也是通常float元素需要手动指定width的原因）

破坏性

文字之所以会环绕含有float属性的图片时因为浮动破坏了正常的line boxes。

在CSS，所有的高度都是由两个模型产生的：

  • box盒模型 对应CSS为height+padding+margin，受height及img的height的值控制
  • line box 对应样式为line-height，而float而之所以能实现浮动环绕，就是因为它破坏了元素的inline boxes，元素已经没有了高度（高度塌陷）
关于文档流

常说的绝对完全脱离文档流，浮动不完全脱离文档流，而这造成的差异就是：同处于文档流的元素实体不会与浮动元素重叠，而会与绝对定位重叠。这就是文字围绕的重要原因之一：虽然浮动元素高度为零，但是因为宽度实体存在（包裹性），同样是content area 实体的文字不会与之重叠（外部的容器盒子containing box会重叠）

关于利用浮动布局

属性值

left 左侧

right 右侧

none 不进行浮动

inline-start 元素必须浮动在其所在块容器的开始一侧

inline-end 元素必须浮动在其所在块容器的结束一侧

如何定位

当一个元素浮动之后，它会被移出正常的文档流，然后向左或者向右平移，一直平移直到碰到了所处的容器的边框，或者碰到另外一个浮动的元素。

关于JS

float是JS的保留关键字，所以想用JS改变float属性不能obj.style.float="left";

清除浮动

目的

解决高度塌陷

为什么会高度塌陷？什么时候会高度塌陷？塌陷原因是：元素含有浮动属性 – 破坏inline box – 破坏line box高度 – 没有高度 – 塌陷。什么时候会塌陷：当标签里面的元素只要样子没有实际高度时会塌陷。所以呢，并不是只要有浮动元素就会坍塌，就要清除的，CSS水平高低衡量的标准之一就是改用什么样式就用什么样式，不多用也不少用。

clear:both

直接一个<div style="clear:both;"></div>放到当作最后一个子标签放到父标签那儿，兼容性强，但是产生了一个空标签，而且不小心多打了一个空格键就会产生高度，所以，不推荐

overflow+zoom方法

代码简洁涵盖几乎所有浏览器，但是，显然并不适合当里面的元素需要负值定位的时候

after+zoom

最推荐，不会影响任何其他样式，通用性强，覆盖面广

display：inline-block

float的局限性

每个列表元素的高度必须要一致，否则就会像是俄罗斯方块一样，“锯齿相错”，即如果第一行有一个列表高度高于其他列表，那就在第二行，第一个元素会沿着最高元素的右侧对齐，因为float破坏了inline box

inline-block

简单而言，就是既有block元素能设置宽高的属性，右像inline一样不换行

间隙

使用display:inline-block会经常出现换行/空格符间隙

最基础的font-size:0字体大小为0

或者letter-spacing:-4px改变文字水平间距，但是受浏览器及字体影响较大，IE7一下更是emm
