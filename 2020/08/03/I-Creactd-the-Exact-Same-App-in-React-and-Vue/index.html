怎样改变数据？

首先，改变数据（mutate data）是什么意思，听起来有点专业？这意味着改变我们存储的数据，例如如果要修改某人的名字值从“John”到“Tom”，我们就改变了数据（mutate the data）。而这也是React和Vue的区别，Vue本质上创建了一个对象用来自由更新数据，而React通过状态钩子来处理。

在React里——至少从2019年以后——我们通常会使用Hooks来处理状态，如果你没见过这种类型的概念它可能有点奇怪，但是基本上，它的工作原理如下：

假设我们想要创建一个待办事项列表。我们可能需要创建一个名为list的变量，它可能会接受一个字符串或数组或者对象(如果我们想给每个todo字符串一个ID或其他一些东西)。我们可以通过写入const [list, setList] = useState([])来设置它。这里我们使用的是一个叫做useState的钩子，它让我们在组件中保持本地状态。

另外，你可能已经注意到我们在useState()中传入了一个空数组[]。我们在里面放的是我们想要的列表初始值。本例中我们想要一个空数组。但是你也可以在接下来的图中看到我们在数组中传递了一些数据，这些数据最终成为list的初始化数据。想知道setList是做什么的？

在Vue中，通常会将组件的所有可变数据放置在setup()函数内，该函数返回一个对象，其中包含要公开的数据和函数（也就是需要在应用中使用的东西）。 可以看到程序中的每个状态数据（也就是我们希望能进行改变的数据）都包装在ref()函数内部。 这个ref()函数是我们从Vue导入的，只要这些数据中的任何一项发生了改变，我们的程序都会更新。 简而言之，如果想在Vue中创建可变数据，请为ref()函数分配一个变量，并将默认数据放入其中。

怎样在应用中引用可变数据

假如我们有名为name的数据被赋值为sunil，

React

在React中，会通过useState()创建类似const [name, setName] = useState('Sunil')的代码片段来声明state。在应用中，我们会通过调用名称来引用相同的数据。这里的关键区别是我们不能简单地写name = 'John'，因为React有相应的限制来防止这种简单的，无顾虑的突变（easy, care-free mutation-making）。在React中，我们会写setName('John')，这就是setName位发挥作用的地方。在const [name, setName] = useState('Sunil')中，他创建了两个变量，其中一个变成了const name = 'Sunil'，而第二个const setName被分配为一个函数，该函数允许赋予新值。

Vue

在Vue中，它位于setup()函数内，并被称为const name = ref(Sunil')。在我们的应用中，我们会通过调用name.value来引用它。对于Vue，如果希望使用ref()函数内部创建的值，则需要在变量上查找.value，而不是简单地调用该变量。换句话说，如果我们想要保存状态的变量的值，我们需要查找name.value，而不是名字。如果要更新name的值，可以通过更新name.value来实现。例如，假设我想把我的名字从Sunil改为John，则需要name.value = "John"。

实际上React和Vue在做同样的事情，即创建可更新的数据。每当更新包装在ref()函数内的数据时，Vue实际上在默认情况下结合了自己版本的name和setName。React要求调用带有内部值的setName()来更新状态，Vue假设你试图更新数据对象内部的值。那么，为什么还要费力地将值从函数中分离出来呢？为什么还需要useState()呢？本质上，React希望在状态发生变化时能够重新运行某些生命周期钩子。在我们的示例中，如果调用setName()， React将知道某些状态已经更改，因此可以运行那些生命周期钩子。如果你直接改变状态，React将不得不做更多的工作来跟踪变化以及运行什么生命周期钩子等等。
